Best Practices

Introduction

Following established best practices helps maintain code quality, improves readability, and reduces bugs. This guide outlines recommended approaches for writing clean, efficient, and maintainable code.

Code Style and Formatting

Consistent Naming Conventions

Use clear, descriptive names for variables, functions, and classes. Be consistent with your naming style throughout the codebase.

DO
// JavaScript const getUserProfile = async (userId) => { const userProfile = await fetchUserData(userId); return userProfile; }; // Python def calculate_average_score(scores): total = sum(scores) return total / len(scores)
DON'T
// JavaScript const getUD = async (id) => { const data = await fetch(id); return data; }; // Python def calc(s): t = sum(s) return t / len(s)

Proper Indentation and Formatting

Maintain consistent indentation and formatting to improve code readability.

DO
if (condition) { doSomething(); if (anotherCondition) { doSomethingElse(); } }
DON'T
if (condition) { doSomething(); if (anotherCondition) { doSomethingElse(); } }

Code Organization

Single Responsibility Principle

Each function or class should have a single responsibility and do one thing well.

DO
// Separate functions for different responsibilities function validateUserInput(input) { // Validation logic } function saveUserData(userData) { // Data saving logic } function notifyUser(message) { // Notification logic }
DON'T
// One function doing multiple unrelated things function processUser(input) { // Validate input if (!input.email || !input.name) { throw new Error('Invalid input'); } // Save to database database.save(input); // Send email notification emailService.send({ to: input.email, subject: 'Account Updated', body: 'Your account has been updated' }); }

DRY (Don't Repeat Yourself)

Avoid code duplication by extracting repeated logic into reusable functions or classes.

DO
function formatCurrency(amount) { return `$${amount.toFixed(2)}`; } const price1 = formatCurrency(10.5); const price2 = formatCurrency(25.75); const price3 = formatCurrency(8.99);
DON'T
const price1 = `$${(10.5).toFixed(2)}`; const price2 = `$${(25.75).toFixed(2)}`; const price3 = `$${(8.99).toFixed(2)}`;

Error Handling

Proper Exception Handling

Handle exceptions appropriately and provide meaningful error messages.

DO
try { const data = JSON.parse(jsonString); processData(data); } catch (error) { console.error('Failed to parse JSON data:', error.message); // Handle the error appropriately showUserFriendlyError('We couldn\'t process your request. Please try again.'); }
DON'T
// No error handling const data = JSON.parse(jsonString); processData(data); // Or catching errors without proper handling try { const data = JSON.parse(jsonString); processData(data); } catch (error) { console.log('Error'); }
